# ML pdf 폴더

이 폴더는 데이터 분석을 위한 PDF 자료를 저장하는 폴더입니다.

## 사용 방법

### 1. PDF 파일 추가
이 폴더에 분석 자료 PDF 파일을 복사하거나 이동하세요.

**예시:**
- `선형 모델.pdf`
- `머신러닝 가이드.pdf`
- `데이터 분석 방법론.pdf`

### 2. 서버 시작
RAG 서버를 시작하면 자동으로 이 폴더의 모든 PDF를 스캔하고 벡터화합니다.

```bash
pnpm run rag-server
```

### 3. 자동 처리 과정
서버 시작 시 다음과 같은 과정이 자동으로 수행됩니다:

1. `ML pdf` 폴더의 모든 PDF 파일을 스캔
2. 각 PDF를 텍스트로 변환하고 청크 단위로 분할
3. 각 청크를 벡터로 변환하여 벡터 DB에 저장
4. 이미 처리된 파일은 스킵하고, 새로 추가된 파일만 처리
5. 처리 결과는 `storage/vectors/vectors.json`에 저장

**콘솔 출력 예시:**
```
3개의 PDF 파일을 발견했습니다. 벡터화를 시작합니다...
처리 중: 선형 모델.pdf
✓ 선형 모델.pdf 처리 완료 (15개 청크)
처리 중: 머신러닝 가이드.pdf
✓ 머신러닝 가이드.pdf 처리 완료 (23개 청크)

PDF 초기화 완료:
  - 새로 처리된 파일: 2개
  - 이미 처리된 파일: 1개
  - 총 벡터 수: 38개
```

### 4. 데이터 분석 사용
1. Load Data 모듈에서 데이터를 로드
2. Properties 패널의 **"AI로 데이터 사전 분석하기"** 버튼 클릭
3. 모달이 열리면 자동으로 이 폴더의 PDF 자료를 기반으로 분석 수행

## 주의사항

- ✅ **PDF 파일만** 이 폴더에 넣어주세요
- ✅ 파일명에 특수문자가 많으면 문제가 발생할 수 있으니 **간단한 이름**을 사용하세요
- ✅ 서버를 재시작하면 **새로 추가된 PDF가 자동으로 처리**됩니다
- ✅ 이미 처리된 PDF는 다시 처리하지 않으므로 **서버 시작이 빠릅니다**

## 폴더 구조

```
ML Auto Flow/
├── ML pdf/              ← 여기에 PDF 파일을 넣으세요
│   ├── 선형 모델.pdf
│   ├── 머신러닝 가이드.pdf
│   └── README.md
└── storage/
    └── vectors/         ← 벡터 DB가 저장되는 곳
        └── vectors.json
```

## 문제 해결

**Q: 서버 시작 시 PDF가 처리되지 않아요**
- PDF 파일이 `ML pdf` 폴더에 있는지 확인하세요
- 파일 확장자가 `.pdf`인지 확인하세요
- 서버 콘솔의 오류 메시지를 확인하세요

**Q: 새 PDF를 추가했는데 분석에 반영이 안 돼요**
- 서버를 재시작하세요 (새 파일은 서버 시작 시에만 처리됩니다)
