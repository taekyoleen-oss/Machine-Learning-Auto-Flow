## 1. 요약 (Executive Summary)

- **과제 유형:** 이진 분류(Binary Classification) — 한 개인의 연 소득이 **\$50,000 초과(`>50K`)인지 이하(`<=50K`)인지** 예측.
- **데이터:** UCI Machine Learning Repository의 **Adult(Census Income)** 데이터셋. **32,561행 × 15열**(특성 14 + 타깃 1).
- **클래스 불균형:** `<=50K` 24,720건(75.9%) vs `>50K` 7,841건(24.1%) — **약 3:1 불균형**. 이 점이 평가지표 해석의 핵심.
- **모델:** **의사결정나무(Decision Tree)**, `criterion=gini`, `max_depth=8`. 입력은 **수치형 6개 특성**(age, fnlwgt, education-num, capital-gain, capital-loss, hours-per-week).
- **분석 흐름:** `LoadData → SplitData(80/20) → DecisionTree(정의) → TrainModel → ScoreModel → EvaluateModel`.
- **핵심 결과(테스트셋 n≈6,513, 임계값 0.50):** 정확도 **0.8320**, 정밀도 **0.8126**, 재현율 **0.3947**, F1 **0.5313**, **ROC-AUC 0.8548**.
- **시사점:** 정확도는 높아 보이지만 이는 다수 클래스(`<=50K`) 비중이 커서 나타나는 착시가 있으며, 소수 클래스(`>50K`) **재현율이 0.39로 낮음**. **임계값(threshold)을 낮추면 재현율을 크게 끌어올릴 수 있어**(0.20에서 재현율 0.81), 목적에 맞는 임계값 선택이 중요하다.

> **강조:** "정확도 83%"라는 단일 숫자만 보면 좋아 보이지만, 불균형 데이터에서는 **AUC·정밀도·재현율·임계값별 곡선**을 함께 봐야 모델을 올바르게 이해할 수 있다. 본 보고서 5~6장이 이 부분을 다룬다.

## 5. 분석 진행 과정 (파이프라인)

본 예제는 6개 모듈이 순서대로 연결된 파이프라인이다. 각 모듈의 역할·파라미터·입출력을 단계별로 설명한다.

```
[1] LoadData ──→ [2] SplitData ──┬─(train)→ [4] TrainModel ──→ [5] ScoreModel ──→ [6] EvaluateModel
                                  │              ↑
                  [3] DecisionTree ─(모델 정의)──┘
                                  └─(test)──────────────────────→ (ScoreModel/Evaluate에 사용)
```

### 5.1 [1] LoadData — 데이터 적재
- **역할:** `adult.csv`(32,561행)를 표 형태(DataFrame)로 읽어 들인다.
- **파라미터:** `source = "adult.csv"`.
- **출력:** 15개 컬럼 전체를 담은 데이터.

### 5.2 [2] SplitData — 학습/평가 분할
- **역할:** 데이터를 **학습용**(train)과 **평가용**(test)으로 나눈다. 모델은 학습 데이터로만 배우고, 한 번도 보지 않은 평가 데이터로 성능을 측정해야 **과적합 없는 일반화 성능**을 알 수 있다.
- **파라미터:** `train_size=0.8`(80% 학습 / 20% 평가), `random_state=42`(난수 고정 → **재현성**), `shuffle=True`, `stratify=False`.
- **결과 규모:** 학습 약 **26,048행**, 평가 약 **6,513행**.

> `random_state=42`로 분할 난수를 고정하기 때문에 **몇 번을 실행해도 같은 분할·같은 결과**가 나온다. 이는 본 앱의 재현성 원칙의 핵심이다.

### 5.3 [3] DecisionTree — 모델 정의
- **역할:** 학습에 사용할 **의사결정나무 분류기**의 구조(하이퍼파라미터)를 정의한다(이 단계는 아직 학습하지 않고 "설정"만 만든다).
- **파라미터:**
  - `model_purpose = classification` — 분류용.
  - `criterion = gini` — 분기 품질 척도로 **지니 불순도(Gini impurity)** 사용. 한 노드가 한 클래스로 순수할수록 불순도가 낮다.
  - `max_depth = 8` — 나무의 최대 깊이를 8로 제한(과적합 방지·해석 가능성 확보).
  - `min_samples_split = 2`, `min_samples_leaf = 1` — 분기/말단 노드의 최소 표본 수(기본값).

> **의사결정나무란?** 데이터를 "예/아니오" 질문(예: `capital-gain > 5000?`)으로 반복 분할하여, 각 말단(leaf)에서 다수 클래스로 예측하는 모델이다. 규칙이 사람이 읽을 수 있어 **해석 가능성**이 높고, 수치 특성의 비선형 관계도 잡아낸다. 깊이를 제한(`max_depth=8`)하면 일반화가 좋아진다.

### 5.4 [4] TrainModel — 학습
- **역할:** 학습 데이터로 의사결정나무를 **실제로 적합**(fit)시킨다.
- **파라미터:**
  - `feature_columns = [age, fnlwgt, education-num, capital-gain, capital-loss, hours-per-week]` — 입력 특성 6개(수치형).
  - `label_column = income` — 예측 대상.
- **출력:** 학습된 모델(분기 규칙 + 각 특성의 중요도).

### 5.5 [5] ScoreModel — 예측(채점)
- **역할:** 학습된 모델로 데이터의 각 행에 대해 예측을 생성한다.
- **출력 컬럼:**
  - `Predict` — 예측된 클래스(하드 라벨, `<=50K` 또는 `>50K`).
  - `income_Predict_Proba_0`, `income_Predict_Proba_1` — 각 클래스의 **예측 확률**. 양성 클래스(`>50K`) 확률(`..._Proba_1`)이 임계값 조정·ROC 곡선의 근거가 된다.

### 5.6 [6] EvaluateModel — 평가
- **역할:** 실제값(`income`)과 예측(`Predict`/확률)을 비교해 분류 성능을 계산한다.
- **파라미터:** `model_type=classification`, `label_column=income`, `prediction_column=Predict`, `threshold=0.5`.
- **산출물:** 정확도/정밀도/재현율/F1, 혼동행렬, **임계값 스윕 표**, **ROC 곡선·AUC**, **정밀도-재현율(PR) 곡선·평균정밀도** — 6장에서 상세히 해석한다.

## 6. 분석 결과와 해석

아래 수치는 평가용 테스트셋(**n ≈ 6,513**, 학습에 쓰이지 않은 데이터)에서 측정한 값이다.

### 6.1 기본 성능 지표 (임계값 0.50)

| 지표 | 값 | 의미 |
|------|----|------|
| Accuracy(정확도) | **0.8320** | 전체 중 맞춘 비율 |
| Precision(정밀도) | **0.8126** | `>50K`로 예측한 것 중 실제 `>50K` 비율 |
| Recall(재현율) | **0.3947** | 실제 `>50K` 중 모델이 잡아낸 비율 |
| F1-Score | **0.5313** | 정밀도·재현율의 조화평균 |
| ROC-AUC | **0.8548** | 임계값 전반의 분리 성능(1에 가까울수록 우수) |
| Average Precision(PR-AUC) | **0.5364** | 불균형 데이터에서 유용한 PR 곡선 아래 면적 |

### 6.2 혼동행렬 (Confusion Matrix, 임계값 0.50)

|  | 예측: `<=50K` | 예측: `>50K` |
|--|--------------|--------------|
| **실제: `<=50K`** | TN = 4,799 | FP = 143 |
| **실제: `>50K`** | FN = 951 | TP = 620 |

- 실제 `>50K`(고소득) **1,571명 중 620명만** 잡아냄(FN 951명 놓침) → **재현율 0.39**.
- `>50K`로 예측한 763명 중 620명이 실제 고소득 → **정밀도 0.81**(예측이 보수적이라 틀리게 고소득이라 외치는 경우는 적음).
- 즉 이 모델은 **확실할 때만 "고소득"이라 판정**하는 경향(보수적). 고소득자를 폭넓게 찾아내야 하는 목적에는 임계값 하향이 필요.

### 6.3 임계값(Threshold) 조정의 효과 ★

EvaluateModel은 0.00~1.00 구간의 임계값을 바(슬라이더)로 조절하며 지표가 어떻게 변하는지 보여준다(양성 클래스 확률 `income_Predict_Proba_1` 기준). 임계값을 **낮추면** 더 적극적으로 `>50K`라 판정 → 재현율↑·정밀도↓, **높이면** 반대.

| 임계값 | Accuracy | Precision | Recall | 해석 |
|--------|----------|-----------|--------|------|
| 0.20 | 0.7305 | 0.4663 | **0.8103** | 고소득자를 폭넓게 포착(재현율↑) 대신 오탐↑ |
| 0.50 | **0.8320** | 0.8126 | 0.3947 | 기본값(정확도 최대 부근) |
| 0.80 | 0.8277 | **0.9517** | 0.3011 | 매우 확실한 경우만 판정(정밀도↑) |

> **핵심 해석:** "올바른 임계값"은 하나로 정해져 있지 않다. **고소득자를 놓치면 안 되는 목적**(예: 타깃 마케팅 후보 확보)이라면 임계값을 0.2 부근으로 낮춰 **재현율 0.81**을 택하고, **잘못된 고소득 판정 비용이 큰 목적**이라면 임계값을 높여 정밀도를 우선한다. 이 트레이드오프를 한눈에 보여주는 것이 임계값 스윕 표와 다음의 곡선들이다.

### 6.4 ROC 곡선과 AUC

- **ROC 곡선:** 임계값을 0→1로 움직일 때 (거짓양성률 FPR, 참양성률 TPR=재현율) 좌표를 그린 곡선. 좌상단(낮은 FPR·높은 TPR)에 가까울수록 우수.
- **AUC = 0.8548** — 무작위 분류기(AUC 0.5)보다 크게 우수하며, 임의의 고소득자가 임의의 비고소득자보다 높은 점수를 받을 확률이 약 **85%**(=0.85)임을 의미한다. 정확도(0.83)보다 AUC가 모델의 **본질적 분리력**을 더 잘 대변한다.

### 6.5 정밀도-재현율(PR) 곡선

- **Average Precision(PR-AUC) = 0.5364.** PR 곡선은 불균형 데이터에서 ROC보다 보수적·현실적인 평가를 제공한다(양성 비율 24% 대비 유의미하게 높음).

## 7. 재현성 (Reproducibility)

본 예제의 모든 분석은 **검증 가능한 Python으로 재현**된다.

- 캔버스 모듈은 브라우저 내 Python(Pyodide)으로 실행되고, "전체 코드 보기"는 **외부에서 그대로 실행되는 standalone Python**을 내보낸다.
- 모든 난수는 고정(`random_state=42`)되어 **2회 실행 결과가 byte 단위로 동일**하며, 회귀 검증(`npm run verify:pipelines`, 픽스처 `13_book_adult_clf`)이 이를 강제한다.
- EvaluateModel이 내보내는 Python에는 **임계값 스윕 표(0.00~1.00, 21행)·ROC 곡선(21점)·AUC**가 포함되어, 앱에서 본 결과를 사용자의 환경에서 그대로 산출할 수 있다.
- 내보낸 코드 실측 검증: 사다리꼴 적분 **AUC = 0.854369**가 sklearn `roc_auc_score`와 일치, 2회 실행 동일.

## 8. 결론 및 한계

**결론**
- 의사결정나무(깊이 8)는 수치형 6개 특성만으로 **AUC 0.85**의 양호한 분리력을 보인다.
- 기본 임계값(0.5)에서는 정확도(0.83)·정밀도(0.81)는 높지만 **재현율(0.39)이 낮아** 고소득자를 절반 이상 놓친다.
- **임계값을 목적에 맞게 조정**하면(예: 0.2) 재현율을 0.81까지 끌어올릴 수 있어, 분류 의사결정은 단일 임계값이 아니라 **곡선 전체와 목적**을 보고 정해야 한다.

**한계 및 개선 방향**
- 본 예제는 **범주형 특성(직업·학력·혼인상태 등)을 사용하지 않는다.** 이들을 인코딩(예: 원-핫/순서형)해 추가하면 성능 향상 여지가 크다.
- `fnlwgt`(인구조사 가중치)는 개인 소득과 인과적 관련이 약한 표본 가중치로, 특성에서 제외하는 것을 검토할 수 있다.
- 불균형(76:24) 보정을 위해 `class_weight` 조정, 임계값 최적화, 또는 앙상블(RandomForest/Gradient Boosting) 적용을 고려할 수 있다.
