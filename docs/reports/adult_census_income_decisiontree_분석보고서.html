1. 요약 (Executive Summary)

  • 과제 유형: 이진 분류 — 한 개인의 연 소득이 $50,000 초과(>50K)인지 이하(<=50K)인지 예측.
  • 데이터: UCI Adult(Census Income) 데이터셋. 32,561행 × 15열(특성 14 + 타깃 1).
  • 클래스 불균형: <=50K 24,720건(75.9%) vs >50K 7,841건(24.1%) — 약 3:1 불균형. 평가지표 해석의 핵심.
  • 모델: 의사결정나무, criterion=gini, max_depth=8. 입력은 수치형 6개 특성.
  • 분석 흐름: LoadData → SplitData(80/20) → DecisionTree → TrainModel → ScoreModel → EvaluateModel.

5. 모델 개발 과정 (파이프라인)

본 예제는 6개 모듈이 순서대로 연결된 파이프라인이다.

[1] LoadData ─→ [2] SplitData ─┬─(train)→ [4] TrainModel ─→ [5] ScoreModel ─→ [6] EvaluateModel
                               │              ↑
                [3] DecisionTree ─(모델 정의)─┘
                               └─(test)──────────────────────→ (Score/Evaluate에 사용)

[1] LoadData — 데이터 적재

  • 역할: adult.csv(32,561행)를 표(DataFrame)로 읽는다.
  • 파라미터: source = "adult.csv". 출력: 15개 컬럼 전체.

[2] SplitData — 학습/평가 분할

  • 역할: 데이터를 학습용(train)/평가용(test)으로 나눈다. 모델은 학습 데이터로만 배우고, 보지 않은 평가 데이터로 일반화 성능을 측정한다.
  • 파라미터: train_size=0.8, random_state=42(난수 고정 → 재현성), shuffle=True, stratify=False.
  • 결과 규모: 학습 약 26,048행, 평가 약 6,513행.

[3] DecisionTree — 모델 정의

  • 역할: 의사결정나무 분류기의 구조(하이퍼파라미터)를 정의한다(아직 학습 안 함).
  • 파라미터: model_purpose=classification, criterion=gini(지니 불순도), max_depth=8(과적합 방지·해석성), min_samples_split=2, min_samples_leaf=1.
의사결정나무란? 데이터를 "예/아니오" 질문(예: capital-gain > 5000?)으로 반복 분할하여 각 말단(leaf)에서 다수 클래스로 예측한다. 규칙을 사람이 읽을 수 있어 해석성이 높고, 수치 특성의 비선형 관계도 잡는다.

[4] TrainModel — 학습

  • 역할: 학습 데이터로 의사결정나무를 실제로 적합(fit)한다.
  • 파라미터: feature_columns=[age, fnlwgt, education-num, capital-gain, capital-loss, hours-per-week], label_column=income.

[5] ScoreModel — 예측(채점)

  • 출력 컬럼: Predict(예측 클래스), income_Predict_Proba_0/1(클래스별 예측 확률). 양성 클래스 확률(..._Proba_1)이 임계값 조정·ROC 곡선의 근거가 된다.

[6] EvaluateModel — 평가

  • 역할: 실제값(income)과 예측을 비교해 성능을 계산한다.
  • 산출물: 정확도/정밀도/재현율/F1, 혼동행렬, 임계값 스윕 표, ROC 곡선·AUC, PR 곡선·평균정밀도.

6. 분석 결과와 해석

아래 수치는 평가용 테스트셋(n ≈ 6,513, 학습에 쓰이지 않은 데이터)에서 측정했다.

6.1 혼동행렬 (임계값 0.50)

예측 <=50K예측 >50K
실제 <=50KTN = 4,799FP = 143
실제 >50KFN = 951TP = 620
  • 실제 >50K 1,571명 중 620명만 포착(951명 놓침) → 재현율 0.39.
  • >50K로 예측한 763명 중 620명이 실제 고소득 → 정밀도 0.81. 모델이 보수적으로 판정함을 의미.

6.2 임계값(Threshold) 조정의 효과

EvaluateModel은 양성 클래스 확률을 기준으로 0~1 임계값을 슬라이더로 조절하며 지표 변화를 보여준다. 임계값을 낮추면 적극적으로 >50K 판정 → 재현율↑·정밀도↓.

임계값AccuracyPrecisionRecall해석
0.200.73050.46630.8103고소득자를 폭넓게 포착(재현율↑), 오탐↑
0.500.83200.81260.3947기본값(정확도 최대 부근)
0.800.82770.95170.3011매우 확실할 때만 판정(정밀도↑)
핵심 해석 — "올바른 임계값"은 하나가 아니다. 고소득자를 놓치면 안 되는 목적이면 임계값을 낮춰 재현율을 택하고, 잘못된 고소득 판정 비용이 크면 임계값을 높여 정밀도를 우선한다.

6.3 ROC 곡선과 AUC

AUC = 0.8548 — 무작위 분류기(0.5)보다 크게 우수하며, 임의의 고소득자가 임의의 비고소득자보다 높은 점수를 받을 확률이 약 85%(=0.85)임을 의미한다. 정확도(0.83)보다 AUC가 모델의 본질적 분리력을 더 잘 대변한다.

6.4 정밀도-재현율(PR) 곡선

Average Precision(PR-AUC) = 0.5364. 불균형 데이터(양성 24%)에서 ROC보다 보수적·현실적인 평가를 제공한다.

7. 재현성 (Reproducibility)

  • 캔버스 모듈은 브라우저 Python(Pyodide)으로 실행되고, "전체 코드 보기"는 외부에서 그대로 실행되는 standalone Python을 내보낸다.
  • 모든 난수는 고정(random_state=42)되어 2회 실행 결과가 byte 단위로 동일하며, 회귀 검증(npm run verify:pipelines, 픽스처 13_book_adult_clf)이 강제한다.
  • 내보낸 EvaluateModel Python에는 임계값 스윕 표(21행)·ROC 곡선(21점)·AUC가 포함되어 동일 결과를 사용자 환경에서 재현할 수 있다.
  • 실측: 사다리꼴 적분 AUC = 0.854369가 sklearn roc_auc_score와 일치, 2회 실행 동일.

8. 결론 및 한계

결론

  • 의사결정나무(깊이 8)는 수치형 6개 특성만으로 AUC 0.85의 양호한 분리력을 보인다.
  • 기본 임계값(0.5)에서는 정확도·정밀도는 높지만 재현율(0.39)이 낮아 고소득자를 절반 이상 놓친다.
  • 임계값을 목적에 맞게 조정하면 재현율을 0.81까지 끌어올릴 수 있어, 분류 의사결정은 곡선 전체와 목적을 보고 정해야 한다.

한계 및 개선 방향

  • 범주형 특성(직업·학력·혼인상태 등)을 사용하지 않는다. 인코딩해 추가하면 성능 향상 여지가 크다.
  • fnlwgt(표본 가중치)는 개인 소득과 인과적 관련이 약해 제외를 검토할 수 있다.
  • 불균형 보정(class_weight), 임계값 최적화, 앙상블(RandomForest/Gradient Boosting) 적용을 고려할 수 있다.
